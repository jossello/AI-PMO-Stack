Create an executive-style interface called **"AI PMO Dashboard"** for a one-day pilot exploring AI-enabled program management.

This dashboard should help a program manager understand portfolio health, recent decisions, and active experiments at a glance.

Use the existing tables:

* Intake
* Decisions
* Experiments

Design for clarity and screenshots, not production use.

## Section 1: Intake Status

Show active intake items with these fields:

* Title
* Priority
* Status

Sort by:

1. Priority (High first)
2. Status

Filter out completed work.

If available, add a chart showing Intake items grouped by Status.

## Section 2: Recent Decisions

Display recent decisions with:

* Decision Title
* Review Date

Sort newest first.

Show only the most recent decisions.

## Section 3: Experiments

Display experiments with:

* Experiment Name
* Status

Group or sort by Status.

## Interface Design Principles

* Simple and executive-friendly
* Optimized for portfolio visibility
* Minimal clicks
* Read-only by default
* Suitable for screenshots in a GitHub portfolio

Display the following subtitle at the top:

**Organizational Memory → Intelligence Layer → Human & Agent Execution**

This interface demonstrates how AI-enabled PMOs can combine structured execution, organizational memory, and human accountability.

Do not add automations, agents, forms, or workflow actions.
